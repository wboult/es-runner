ES Runner launches official Elasticsearch or OpenSearch ZIP distributions in a
local process, waits for readiness, and provides a small Java API for
interaction. These docs are organized with the **Diataxis** framework so you
can choose the style that fits your goal.

## Start here

- **One local node for tests or tooling**
  Start with [Getting started](tutorials/getting-started/) or
  [First single-node server](tutorials/first-server/).
- **A Gradle build with multiple suites or subprojects**
  Start with [Choose a shared-cluster backend](how-to/choose-gradle-shared-cluster-backend/).
- **OpenSearch instead of Elasticsearch**
  Use [Run OpenSearch](how-to/run-opensearch/).
- **You want the artifact/module map first**
  Use [Module overview](reference/modules/).

## Why this instead of Testcontainers or custom scripts?

ES Runner is aimed at the cases where a real local node is useful but Docker is
not the right default.

- **Use ES Runner** when you want process-backed realism, Docker-free CI, or a
  shared build-scoped cluster for multiple Gradle suites and subprojects.
- **Use Testcontainers** when Docker is already standard and you want
  container-level isolation as the main abstraction.
- **Use custom scripts** only if your lifecycle is so narrow that you do not
  need reusable download, cleanup, diagnostics, or namespacing support.

## Release status

ES Runner already has a tag-driven release pipeline for Maven Central and the
Gradle Plugin Portal, but until the first live release is cut you should treat
composite-build and local-repo consumption as the normal path.

If you want to verify the publication flow now, start with:

- [Release ES Runner](how-to/releasing/)
- [Use shared Gradle test clusters](how-to/gradle-shared-test-clusters/)

## Suggested progression

If you are adopting ES Runner for the first time, this path takes you from a
single local node to a full multi-project CI setup:

1. **Start one node locally** - [Getting started](tutorials/getting-started/)
2. **Customize ports or install plugins** - [Customize ports](how-to/custom-ports/) / [Install plugins](how-to/install-plugins/)
3. **Use the official Java client** - [Official Java Client](how-to/official-java-client/)
4. **Set up mirrors or caching** - [Use a mirror](how-to/use-a-mirror/) / [Reuse a distro](how-to/reuse-pre-downloaded-distro/)
5. **Share one cluster across Gradle suites** - [Shared Gradle clusters](how-to/gradle-shared-test-clusters/)
6. **Choose the right shared-cluster backend** - [Backend choice guide](how-to/choose-gradle-shared-cluster-backend/)
7. **Run it in CI** - [GitHub Actions example](how-to/ci-github-actions/)
8. **Tune performance** - [Performance guidance](explanation/performance/)
